Support Agenta

AI asistent zodpovídá otázky návštěvníků o vašich službách a eskaluje nevyřešené dotazy e-mailem. Nasazení trvá přibližně 10 minut.

Node.js 18+
Účet na 21st.dev
Účet na Resend (pro e-mail)
1
Nasaďte agenta na 21st.dev

Přejděte do složky agenta, nainstalujte závislosti a přihlaste se ke svému 21st.dev účtu pomocí API klíče. Klíč najdete v 21st.dev → Settings (začíná an_sk_).

Terminál
# Přejděte do složky agenta cd agents/support-agent # Nainstalujte závislosti npm install # Přihlaste se (vloží se API klíč ze 21st.dev) npx @21st-sdk/cli login
Nasazení
npx @21st-sdk/cli deploy
Po úspěšném nasazení uvidíte slug agenta: support-agent. Uložte si ho — budete ho potřebovat v dalším kroku.
2
Nastavte proměnné prostředí agenta

Agent potřebuje vědět, odkud stáhnout dokumentaci webu (DOCS_URL), a mít přístup k Resend pro odesílání e-mailů.

Proměnné agenta na 21st.dev
# URL vašeho webu (kde jsou llms.txt a llms-full.txt) npx @21st-sdk/cli env set support-agent DOCS_URL https://financnistrateg.com # API klíč z resend.com (začíná re_) npx @21st-sdk/cli env set support-agent RESEND_API_KEY re_váš_klíč # E-mail, kam se posílají eskalované dotazy npx @21st-sdk/cli env set support-agent SUPPORT_EMAIL adamdvorak@financnistrateg.com # (Volitelné) odesílatel e-mailů npx @21st-sdk/cli env set support-agent RESEND_FROM_EMAIL "Asistent <asistent@financnistrateg.com>"
Znovu nasadit po nastavení proměnných
npx @21st-sdk/cli deploy
Resend API klíč získáte zdarma na resend.com. Bezplatný plán umožňuje 100 e-mailů denně — pro osobní web zcela dostačující.
3
Vyplňte .env.local na serveru

Soubor .env.local v kořeni projektu obsahuje API klíč pro server (serve.mjs), který zajišťuje výměnu tokenů mezi prohlížečem a 21st.dev. Klíč nikdy neposílejte do Gitu — soubor je v .gitignore.

.env.local
# API klíč ze 21st.dev → Settings (an_sk_...) API_KEY_21ST=an_sk_váš_klíč # Resend API klíč (re_...) RESEND_API_KEY=re_váš_klíč # E-mail pro eskalaci dotazů SUPPORT_EMAIL=adamdvorak@financnistrateg.com # URL webu pro stažení llms.txt agentem DOCS_URL=https://financnistrateg.com

Přehled všech proměnných:

Proměnná Kde Popis
API_KEY_21ST Povinné API klíč z 21st.dev/settings
RESEND_API_KEY Povinné API klíč z resend.com pro odesílání e-mailů
SUPPORT_EMAIL Povinné E-mail, kam agent posílá eskalované dotazy
DOCS_URL Povinné URL webu, odkud agent stahuje llms.txt
RESEND_FROM_EMAIL Volitelné Odesílatel e-mailů (výchozí: onboarding@resend.dev)
4
Spusťte server a otestujte

Spusťte lokální server a otevřete web — v pravém dolním rohu uvidíte zlaté chat tlačítko.

Spuštění (z kořene projektu)
node serve.mjs

Ověřte, že token endpoint funguje:

Test tokenu (curl)
curl -X POST http://localhost:3000/api/agent/token
Úspěšná odpověď vypadá takto:
{"token":"sk-...","expiresAt":"2026-05-08T..."}
Chyba "API_KEY_21ST not configured"? Zkontrolujte, že .env.local existuje a obsahuje správný klíč. Server je potřeba restartovat po každé změně .env.local.
5
Nasazení do produkce

Pro produkci nahraďte serve.mjs libovolným Node.js serverem nebo nasaďte na platformu jako Railway, Render nebo VPS. Proměnné prostředí nastavte přímo v platformě (nepoužívejte .env.local v produkci).

Ujistěte se, že soubory llms.txt a llms-full.txt jsou dostupné na URL nastavené v DOCS_URL. Agent je stahuje při startu každé session.

Ověření dostupnosti dokumentace
curl https://financnistrateg.com/llms.txt curl https://financnistrateg.com/llms-full.txt
✦
Asistent je připraven
Zlaté chat tlačítko je nyní aktivní na webu. Návštěvníci se mohou ptát a dostávat odpovědi z dokumentace. Nevyřešené dotazy přijdou e-mailem.
